My rover system simulates a depth sensor that transmits the measured space on the front, left, and right sides of the rover. This data is published by the sensor node as JSON formatted values and processed by the perception node to determine if obstacles are present in any directions. The perception node publishes its data, which is received by the decision node to decide if the next direction of travel is forward, left, or right. This data is received by the motor node to change the linear_x or angular_z velocities. The sensor node publishes to į/scanš. The perception node subscribes to į/scanš and publishes to įobstacle_infoš. The decision node subscribes to į/obstacle_infoš and publishes to įcmd_velš. The motor node subscribes to į/cmd_velš and changes the movement direction. The system uses three topics in total.   This type of data is important to a robotics system because it enables navigation and real-time decision making. By monitoring nearby obstacles, the rover can react dynamically to its environment and avoid collisions without human control. The modular ROS2 node architecture also demonstrates how sensing, perception, decision-making, and motor control can be separated into independent components that communicate through ROS2 topics, which is a design philosophy of this team.
